# PRD — 帕鲁风格全局界面改造

## 背景

PalWorld Server Manager 前端（Next.js 16 + Tailwind v4 + shadcn/ui，静态导出）当前使用 shadcn 默认蓝色主题，配色为通用蓝/紫/粉渐变，缺乏与游戏《幻兽帕鲁》的视觉关联。需要将整个界面改造为**明亮清新卡通**的帕鲁风格，让工具在视觉上与游戏世界呼应。

## 目标

将全站改造为帕鲁风格的「明亮清新卡通」**UI/UX 整体改造**：不仅统一视觉基调（晴空蓝背景、青绿(teal)主色、暖阳橙点缀、圆润卡通卡片与圆润字体，形成全局设计系统），并**按需重构布局结构、信息架构、交互逻辑与数据流**，以呈现更贴合帕鲁风格、体验更佳的界面。

## 用户与场景

- 服务器管理员在桌面浏览器中使用本工具管理帕鲁服务器。
- 主要界面：首页(landing)、服务器管理页、各类弹窗（新增服务器、设置、日志）。

## 范围

### In Scope
1. **设计 token 重构**：在 `ui/src/app/globals.css` 的 `:root` / `.dark` 中，将配色、圆角、边框重铸为帕鲁明亮卡通调色板。
2. **全局字体**：通过 `next/font` 引入圆润无衬线字体（拉丁字形），CJK 走系统回退，保证 zh/ja 正常显示。
3. **氛围层（纯 CSS）**：晴空渐变背景、柔和云朵/山丘感的径向色块、卡通化的"厚"阴影工具类——不引入图片资源，兼容静态导出。
4. **硬编码颜色清扫**：将约 40 处硬编码色值（`from-blue-*`、`text-gray-*`、`from-blue-600 to-purple-600` 等）改为设计 token 或新的帕鲁渐变体系。涉及文件：`layout.tsx`、`page.tsx`、`Navbar.tsx`、`ServerCard.tsx`、`ServerLogsDialog.tsx`、`servers/page.tsx`。
5. **组件卡通化微调**：卡片圆角/边框/悬停动效、按钮质感、Badge 状态色与帕鲁语义色对齐。
6. **布局结构与信息架构重构（按需）**：调整页面骨架、导航、分区、栅格与层级，使信息组织更清晰、更契合帕鲁风格（如首页 hero/feature 结构、服务器页列表/卡片布局、弹窗的分区与信息密度）。
7. **交互逻辑优化（按需）**：加载/空/错误态、状态反馈、动效过渡、操作可发现性与确认流程等交互体验改良。
8. **数据流调整（按需）**：仅在服务于上述 UX 改良时调整前端数据获取/状态管理形态（如查询缓存、加载态呈现、局部刷新），**不改变后端 API 契约**。

### Out of Scope
- **不改变现有功能集**：不新增/删除业务功能，所有既有能力（服务器 CRUD、生命周期、Mod、日志、设置、i18n）保持可用。
- **不改后端**：不改后端 API 形态与契约、数据库、i18n 文案内容（本次为前端 UI/UX）。
- **不破坏架构约束**：保持静态导出（`output: 'export'`）与单文件嵌入分发。
- **不引入外部图片/纹理资源**（保持轻量）。

## 约束

- **静态导出**：`output: 'export'`，无 SSR，图片不优化——氛围装饰必须是 CSS，字体走 `next/font` 自托管。
- **暗色模式保留**：`.dark` class 机制不能破坏，明/暗两套都要成立。
- **多语言**：字体必须对中文/日文有合理回退，不能出现豆腐块或字重丢失。
- **shadcn token 契约**：`@theme inline` 的 token 映射结构保持不变，仅改值，确保所有 primitive 自动继承。
- **平台**：以 Windows 构建为准（项目现状）。

## 验收标准

1. 首页、服务器页、所有弹窗在**亮色**下呈现统一帕鲁明亮卡通风格：晴空蓝背景、青绿主色按钮、暖橙点缀、圆润卡片。
2. **暗色模式**切换后风格自洽（深青蓝底 + 发光点缀），无对比度过低或撞色。
3. 全站**无残留**蓝紫粉旧配色（`from-blue-600 via-purple-600 to-pink-600` 一类）；硬编码色值已全部改为 token 或帕鲁体系。
4. 圆润字体在 zh/en/ja 三语言下均正常显示，无缺字。
5. `bun run build` 成功产出静态导出，`bun run lint` 无新增报错。
6. 关键状态语义色可辨：运行(绿)、停止(灰)、安装中(蓝/信息)、错误(红)、警告(琥珀)。
7. **功能零回退**：重构布局/交互/数据流后，所有既有功能（CRUD、启停、安装、日志 SSE、设置保存、语言切换）仍正常工作。
8. **交互态完整**：加载态、空态、错误态、操作反馈在改造后均有清晰呈现，无裸露空白或无反馈操作。
9. **信息架构更优**：页面分区与导航层级清晰，关键操作可发现、信息密度合理（对照改造前有可感知提升）。
